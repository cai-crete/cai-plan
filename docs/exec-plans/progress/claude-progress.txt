## 방금 무엇을 했는지

### 기능 개요
아트보드(artboard 타입)에 업로드된 배경 이미지(`item.src`)를 "이미지 변환 모드"에서
자유롭게 리사이즈(가로/세로 독립) 및 회전할 수 있는 기능 구현.

### 구현 내역

**types/canvas.ts**
- `CanvasItem`에 `contentTransform?: { x, y, width, height, rotation }` 필드 추가

**page.tsx — 상태/Ref 추가**
- `imageEditingId` 상태: 현재 이미지 변환 모드 중인 아이템 id
- `imageEditingIdRef`: ref mirror
- `imageEditSnapshot`: 편집 진입 시점의 contentTransform (취소 롤백용)
- `isTransformingImage` / `imageTransformOp` / `imageTransformItemId`: 드래그 op 추적
- `imageResizeAxis`: resize 핸들별 dx/dy (-1|0|1)
- `imageTransformStart`: 드래그 시작 시점 스냅샷 (ptX, ptY, tx, ty, tw, th, tr)

**page.tsx — handleReplaceArtboardImage 수정**
- 이미지 교체 시 fit-contain 비율로 `contentTransform` 초기값 자동 계산 후 저장

**page.tsx — 핸들러 추가**
- `handleImageTransformConfirm`: 변환 확정, 히스토리 push, 편집 모드 종료
- `handleImageTransformCancel`: contentTransform 롤백, 편집 모드 종료

**page.tsx — handlePointerDown 수정**
- `img-rotate-handle` 클래스 감지 → rotate op 시작
- `img-resize-handle` 클래스 감지 + data-dx/dy → resize op 시작
- `img-move-area` 클래스 감지 → move op 시작
- 이미 선택된 artboard with src 재클릭 → 이미지 편집 모드 진입
- 편집 모드 중 다른 아이템 클릭 → 편집 확정 후 선택 변경
- 빈 캔버스 클릭 → 편집 확정 후 선택 해제

**page.tsx — handlePointerMove 수정**
- `isTransformingImage` 분기 추가
  - move: contentTransform x/y 업데이트
  - resize: dx/dy 축별 독립 조절, Shift = aspect ratio 고정
  - rotate: atan2 각도 계산, Shift = 15° 스냅

**page.tsx — handlePointerUp 수정**
- `isTransformingImage` 종료 처리 추가

**page.tsx — 렌더링 수정**
- 아이템 div: 편집 모드 시 `overflow: visible` (핸들이 아트보드 경계 밖 표시)
- 이미지: contentTransform 존재 시 absolute 포지셔닝 + rotate CSS 적용
  - 편집 모드 시 `img-move-area` 클래스, `cursor: move`, `pointerEvents: all`
- 이미지 변환 핸들 오버레이 (편집 모드 시):
  - 주황 점선 테두리 (이미지 경계)
  - 8개 resize 핸들 (모서리 4 + 엣지 4), `img-resize-handle` 클래스
  - 1개 rotate 핸들 (이미지 상단 중앙 28px 위), `img-rotate-handle` 클래스
- 선택 테두리: 편집 모드 시 파란색 → 주황색(`#f97316`)
- 상단 컨트롤바(ZoomIn/Out/Lock): 편집 모드 중 숨김
- 상단 floating control bar: 편집 모드 중 숨김
- 하단 확정/취소 바 (편집 모드 시):
  - 아트보드 하단 중앙, 상단 컨트롤바와 동일한 pill 스타일
  - 각도 레이블 (rotation !== 0일 때)
  - ✓ 확정 버튼 (초록), ✕ 취소 버튼 (회색)
