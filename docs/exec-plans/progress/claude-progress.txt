[2026-04-21] sketch-to-plan — N05: Generated Image Aspect Ratio Fix

## 방금 무엇을 했는지

### 문제
생성된 평면도 이미지의 비율이 소스 아트보드와 다르게 출력됨.
3개 레이어에 걸친 버그: API(비율 미전달) + 클라이언트(intrinsic 치수 무시) + CSS(object-cover 크롭)

### 수정 내역

#### 1. route.ts — 아트보드 치수 수신 + 생성 프롬프트에 비율 지시
- 요청 body에 `artboard_width`, `artboard_height` 파라미터 추가
- 두 값이 존재하면 generation prompt에
  "출력 이미지의 가로:세로 비율은 반드시 {W}:{H} (입력 스케치와 동일)이어야 합니다." 삽입

#### 2. usePlanGeneration.ts — PlanGenerationParams에 치수 추가
- `artboardWidth?: number`, `artboardHeight?: number` 필드 추가
- fetch body에 `artboard_width`, `artboard_height` 포함

#### 3. page.tsx — generate() 호출 시 치수 전달
- `handleGenerate`에서 `artboardWidth: item.width, artboardHeight: item.height` 전달

#### 4. page.tsx — 생성 이미지 intrinsic 치수 감지 후 배치
- `generatedPlanImage` effect를 비동기로 변경
- `new Image()` 로 `img.naturalWidth / img.naturalHeight` 감지
- planItem의 width/height를 실제 이미지 치수로 설정 (fallback: sourceArtboardSize)
- canvasItems 대신 `canvasItemsRef.current` 참조 (stale closure 방지)

#### 5. page.tsx JSX — object-cover → object-fill
- `sketch_generated` 이미지 렌더: `object-cover` → `object-fill`
- 컨테이너 = 이미지 intrinsic 치수이므로 크롭 불필요
